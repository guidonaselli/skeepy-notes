| # | When | Scope | Decision | Choice | Rationale | Revisable? |

| D001 | M001 | arch | Desktop framework | Rust + Tauri 2.x | Lowest idle resource usage (~15-40MB RAM, ~0% CPU idle). Electron (~200MB RAM) descartado. .NET WPF válido pero Windows-only sin portabilidad futura. Tauri = WebView2 nativo en Win10/11 + Rust backend sin GC | No |
| D002 | M001 | arch | UI framework | Solid.js | No Virtual DOM = DOM mutations directas, bundle mínimo. React descartado por overhead. Svelte también válido — Solid elegido por reactividad granular más predecible | Yes — si Svelte demuestra mejor DX en equipo |
| D003 | M001 | data | Local storage | SQLite con WAL mode + FTS5 | Zero-config, single file, FTS5 nativa para search <50ms en 10k notas, WAL elimina fsync por escritura. PostgreSQL/Redis descartados (sin servidor). | No |
| D004 | M001 | security | Token storage | Windows Credential Manager (DPAPI) via keyring crate | Tokens cifrados con clave del usuario del OS. NUNCA en SQLite ni archivos de config. | No |
| D005 | M001 | arch | Provider model | Trait-based plugin architecture (NoteProvider trait) | Desacoplamiento total UI ↔ provider. Keep es un plugin, no el core. Si Google depreca la API, la app sigue funcional. | No |
| D006 | M001 | scope | Sync V1 strategy | Pull-only / read-only | Sin escritura a providers en V1. Simplifica enormemente el modelo de conflictos. Local storage es source of truth. | Yes — V2 agrega write support |
| D007 | M001 | arch | Async runtime | Tokio | Runtime async estándar en Rust. Sin threads bloqueantes en sync engine. Concurrencia eficiente sin polling. | No |
| D008 | M001 | integration | Google Keep API | notes.googleapis.com/v1 (oficial, Mayo 2021) | API oficial con OAuth2. Scope keep.readonly suficiente para V1. No scraping, no browser automation. Limitaciones: sin imágenes/audio, sin PATCH, sin webhooks — solo pull. | Yes — revisar si Google depreca |
| D009 | M001 | scope | Keep provider stability | ProviderStability::Experimental | Google tiene historial de deprecar productos. Keep no tiene SLA público de API. Marcar como Experimental hasta 2+ años de track record post-launch. | Yes — upgrade a Stable con evidencia |
| D010 | M001 | product | V1 scope | App residente + local provider + Keep read-only + FTS5 search + layout persistente + autostart | Sin editor inline, sin write a providers, sin auto-update, sin firma de código, sin provider Markdown. | No |
| D011 | M001 | dist | Windows autostart | HKCU registry (no admin required) | HKCU\Software\Microsoft\Windows\CurrentVersion\Run. Sin UAC prompt. Win10/11 compatible. | No |
| D012 | M001 | ux | Close behavior | Minimize to tray (no exit) | App residente no debe cerrarse con X. Exit real solo via menú del tray. | No |
| D013 | M001 | observability | Telemetry | Desactivada por defecto, opt-in explícito | Privacy-first. No hay argumento suficiente para telemetría automática en V1. | Yes — V2 si hay caso de uso justificado |
| D014 | M001 | dist | License | MIT | Más permisiva, menor fricción para adopción open source. Alternativa: MIT OR Apache-2.0 dual (protección de patentes). | Yes — antes de V1 public release |
| D015 | M001 | data | Min Windows version | Windows 10 | WebView2 preinstalado desde Win10. Sin soporte para Win7/8 (EOL). Documentar claramente. | No |
| D016 | M001 | dist | OAuth client_id model | UN client_id del developer compilado en el binario via env vars | OAuth2 es gratis sin importar la cantidad de usuarios. No se necesita backend. El PKCE flow (ya implementado) es el modelo oficial de Google para "Installed Applications". Cada usuario guarda SUS tokens en SU Credential Manager — el dev nunca toca datos de usuarios. client_id se inyecta en build via `env!("GOOGLE_CLIENT_ID")` y GOOGLE_CLIENT_SECRET como GitHub Actions secrets. | No — salvo que Google cambie su política |
| D017 | M001 | ux | BYO credentials override | Campo opcional en Settings para que power users usen su propio Google Cloud project | El 99% de usuarios no lo necesita. El 1% power user (ej: admins corporativos, devs) puede bypasear el client_id default si tiene su propio project de Google. Persiste en settings table bajo claves "keep_client_id" / "keep_client_secret". Si vacío, usa el compilado en el binario. | Yes — puede eliminarse si causa confusión |
| D018 | M001 | dist | Microsoft Store | Post-V1 — MSIX packaging via WiX bundler (Tauri 2.x) | Store requiere MSIX no NSIS. Tauri lo soporta. Necesita cuenta Partner Center (~$20 one-time). Certificación tarda 1-3 días. No requiere cambios en el OAuth flow ni en la arquitectura. `bundle.targets` agrega `["nsis", "msi"]`. | Yes — cuando la app alcance madurez suficiente para Store |
| D019 | M001 | ux | Icon source | logo.svg como fuente canónica, generada via `npx @tauri-apps/cli icon logo.svg` | El CLI de Tauri usa resvg internamente — el rasterizador SVG más fiel. Genera todos los tamaños requeridos (ICO, PNG RGBA 32x32/128x128, AppX, ICNS) en un solo comando. NUNCA editar los archivos en `icons/` a mano — siempre regenerar desde el SVG. | No |
| D020 | S21 | dist | Auto-update strategy | tauri-plugin-updater + GitHub Releases + latest.json | El workflow de release genera y sube latest.json firmado con minisign. La app chequea en background 30s post-startup y emite evento update://available si hay versión nueva. Keys: PUBLIC en tauri.conf.json plugins.updater.pubkey, PRIVATE en secret TAURI_SIGNING_PRIVATE_KEY. | No |
| D021 | S21 | ux | NoteCard visual theme | Windows Sticky Notes light palette | Cambio de dark (#2d2d3d) a light (#fff9c4 amarillo default). Header strip más saturado, texto dark (#1a1a1a). Idéntico a la app de notas nativa de Windows. | Yes — si cambia el diseño |
| D022 | S42 | arch | Popout window model | Una WebviewWindow por nota, sin decoraciones, label = `note-{uuid}` | Cada nota puede existir simultáneamente en el manager (NoteGrid) y como ventana flotante. `main.tsx` enruta por `?note=ID` query param. Estado de visibilidad persiste en `NoteLayout.visible`. Startup restore reabre todas las notas visibles. | No |

Emit desde `note_window_show` y desde `on_window_event(CloseRequested)`. | No |
